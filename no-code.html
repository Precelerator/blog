No-Code Tools: Beispiele & Anwendungen

By Fabio Maienschein
29/01/2021

Die Idee hinter No-Code Tools ist so simpel wie mächtig: Nicht-technische Nutzer werden durch konfigurierbare (oft grafische) Editoren befähigt, selbstständig Anwendungen für sich und andere bauen zu können. Die Vision ist es, dass Fachexperten Anwendungen für ihre eigenen Prozesse und Anwendungsfälle entwickeln können, ohne auf Unterstützung durch Programmierer angewiesen zu sein.

Dieser Vision haben sich in den letzten Jahren viele schnell wachsende Startups verschrieben, aber auch die großen Player reagieren: Microsoft veröffentlichte mit der Power Platform eine No-Code Plattform, die sich fließend mit den großen Microsoft-Systemen integriert (Dynamics), AWS veröffentlichte mit Honeycode eine Konkurrenzlösung, und auch Google investiert in aufstrebende No-Code Anbieter wie z.B. AppSheet.

No-Code Tools erlauben durch die geringere Einstiegshürde (im Vergleich zu traditioneller Programmierung) eine schnelle Einarbeitung und infolge schnelle Iterationen beim Ausarbeiten einer Idee. Anhand simpler Prototypen können Ideen mit wenig Anlaufzeit ausgetestet werden und frühzeitig erstes User-Feedback eingeholt werden. Gerade für Proof-of-Concept Prototypen und simple/generische Anwendungsfälle gibt es bereits eine Reihe möglicher No-Code Tools, die versuchen diese Anwendungsfälle abzudecken. No-Code ist somit nicht zwingend ein Ersatz für Programmierung, sondern eher ein weiteres Werkzeug zur Erstellung von Anwendungen. Gerade für nicht-technische Nutzer bieten No-Code Tools die Möglichkeit, sich ein grundlegendes Verständnis zum Aufbau und zur Gestaltung von Anwendungen anzueignen, ohne zwingend Programmieren lernen zu müssen.

Besonders beliebt sind No-Code Tools für simple Benutzeroberflächen sowie zur Modellierung einfacher Prozesse und Workflows. In der folgenden Übersicht findet sich eine Auswahl von Tools gegliedert nach typischen No-Code Anwendungsbereichen. Eine ausführlichere Liste findet sich hier oder hier.

Landing PagesCarrd, Launchaco
WebseitenWix, Webflow, Wordpress (+ Elementor etc.)
Web ShopsShopify, Wordpress + WooCommerce, Gumroad
App-EntwicklungGlideapps, AppSheet, AppGyver, Adalo, Bubble
ChatbotsLandbot (Bot), Tawk.to (Live Chat)
Integration von ServicesZapier, Integromat
Voice InterfacesVoiceFlow
AI/MLTeachable Machine
Hardware/ Eventbasierte SystemeNode-RED

Zum Einstieg in No-Code Apps lohnt sich ein Blick auf Glideapps: Das Tool erlaubt es einem, basierend auf einem Google Sheet als Datenquelle eine Progressive Web App (PWA) zu generieren, die man mit Nutzern bequem per URL oder QR-Code teilen kann. Eine Einführung in Glide findet sich im hier verlinkten Blogpost.

Fazit

No-Code Tools verbergen technische Implementierungsdetails vom Anwender und erleichtern die Erstellung von Anwendungen für nicht-technische Nutzer ungemein. Da das Endergebnis eine lauffähige Anwendung ist (und nicht der Quellcode), begibt man sich allerdings in eine Abhängigkeit vom Betreiber der No-Code Plattform, die für Prototypen nicht weiter problematisch ist, aber bei geschäftskritischen Anwendungen sorgfältig bedacht werden sollte.

Da No-Code als Trend weiter Fahrt aufnimmt, bleibt abzuwarten, welche Anbieter und Betreiber sich langfristig durchsetzen werden. Es lohnt sich, anhand von Beispiel-Anwendungen Features und Integrationen einer Plattform zu evaluieren, aber durch großzügige Free-Plans können quasi alle Tools problemlos ausgetestet werden, um einen Abgleich mit den Anforderungen des eigenen Anwendungsfalls vornehmen zu können. Nicht alle No-Code Plattformen unterstützen fortgeschrittene Features wie Versionierung oder Kollaboration, was sich in den nächsten Jahren aber weiter verbessern sollte. Gerade in der Anwendungserstellung ist ein gewisses technisches Verständnis für No-Code Tools trotzdem relevant, um abschätzen zu können, welche Entscheidungen einem das jeweilige Tool abnimmt und welche Art von Anwendung das Tool am Ende produziert (Web-App? Native App? Responsive? etc.).
